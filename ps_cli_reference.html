Use with the studio: the dynamic PS Config Factory CLI and rollout map inside index.html already fills in the paths and role context from your topology. This page is the generic shell cookbook.

1. Token legend

Replace these placeholders before running commands:

  • <app> = customer-prefixed app name from the PS Config Factory
  • <file> = generated conf file downloaded from the studio
  • <cm_fqdn> = cluster manager management address
  • <sh1_fqdn> = one SHC member management address
  • <splunk_home> = usually /opt/splunk
  • <splunkforwarder_home> = usually /opt/splunkforwarder

2. Placement map CLI quick start

Deployment Server path

mkdir -p <splunk_home>/etc/deployment-apps/<app>/local
cp <file> <splunk_home>/etc/deployment-apps/<app>/local/
<splunk_home>/bin/splunk reload deploy-server
<splunk_home>/bin/splunk list deploy-clients

Cluster Manager path

mkdir -p <splunk_home>/etc/manager-apps/<app>/local
cp <file> <splunk_home>/etc/manager-apps/<app>/local/
<splunk_home>/bin/splunk validate cluster-bundle --check-restart
<splunk_home>/bin/splunk apply cluster-bundle --answer-yes
<splunk_home>/bin/splunk show cluster-bundle-status

SHC deployer path

mkdir -p <splunk_home>/etc/shcluster/apps/<app>/local
cp <file> <splunk_home>/etc/shcluster/apps/<app>/local/
<splunk_home>/bin/splunk show shcluster-status --verbose -auth <admin>:<password>
<splunk_home>/bin/splunk apply shcluster-bundle -target https://<sh1_fqdn>:8089 -auth <admin>:<password>

Local app path

mkdir -p <splunk_home>/etc/apps/<app>/local
cp <file> <splunk_home>/etc/apps/<app>/local/
<splunk_home>/bin/splunk btool <confname> list --debug
<splunk_home>/bin/splunk restart

3. Deployment client and forwarding baselines

Set deploy-poll on an Enterprise node

<splunk_home>/bin/splunk set deploy-poll <deployment_server_fqdn>:8089 -auth <admin>:<password>
<splunk_home>/bin/splunk restart

Set deploy-poll on a Universal Forwarder

<splunkforwarder_home>/bin/splunk set deploy-poll <deployment_server_fqdn>:8089 -auth <admin>:<password>
<splunkforwarder_home>/bin/splunk restart

Validate outputs, inputs, and deploymentclient settings

<target_home>/bin/splunk btool outputs list --debug
<target_home>/bin/splunk btool inputs list --debug
<target_home>/bin/splunk btool deploymentclient list --debug

4. Indexer-cluster and indexer-discovery references

Peer registration example

<splunk_home>/bin/splunk edit cluster-config -mode peer   -manager_uri https://<cm_fqdn>:8089   -replication_port 9887   -secret <idxc_secret>

Forwarder indexer-discovery example

[tcpout]
defaultGroup = default-autolb-group

[tcpout:default-autolb-group]
indexerDiscovery = idxc1
useACK = true

[indexer_discovery:idxc1]
manager_uri = https://<cm_fqdn>:8089
pass4SymmKey = <discovery_secret>

Peer-side validation

<splunk_home>/bin/splunk show cluster-status --verbose
<splunk_home>/bin/splunk btool server list --debug
<splunk_home>/bin/splunk btool indexes list --debug

7. Troubleshooting CLI pack

Generic health pack

<target_home>/bin/splunk btool check
<target_home>/bin/splunk diag --collect=etc,var/log/splunk

Log triage

tail -n 120 <splunk_home>/var/log/splunk/splunkd.log
egrep 'ERROR|WARN' <splunk_home>/var/log/splunk/splunkd.log | tail -50

Network and listener checks

ss -lntp | egrep '8000|8088|8089|9997|9887|8181|8191'
curl -k https://<target_fqdn>:8089/services/server/info -u <admin>:<password>

Config precedence check

<target_home>/bin/splunk btool server list --debug
<target_home>/bin/splunk btool indexes list --debug
<target_home>/bin/splunk btool props list --debug
<target_home>/bin/splunk btool transforms list --debug

9. Operator reminders

  • Copy the generated file from the PS Config Factory first, then stage it with the CLI.
  • Validate on the real winning tier with btool.
  • Use the same repository for rollback that you used for deployment.
  • Record the exact commands and validation results in the handover note.
